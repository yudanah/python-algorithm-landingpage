## 1. 프로젝트 개요

### 1.1 목적
코딩 학원, 학교 교사, 방과후 교실 담당자를 대상으로 **Let's Coding 파이썬 학습 플랫폼**의 핵심 가치를 전달하고, **도입 문의**로 연결하기 위한 오프라인 홍보 리플릿.

### 1.2 배포 상황
- 코딩 교육 박람회 / 에듀테크 전시회 부스
- 학원 원장 대상 영업 미팅 시 핸드아웃
- 학교 방과후 교실 담당 교사에게 우편/방문 전달
- 학부모 설명회 시 배포

### 1.3 핵심 타깃 (우선순위순)

| 순위 | 대상 | 관심사 |
|------|------|--------|
| 1 | 학원 원장 / 코딩 교육 담당자 | 체계적 커리큘럼, 학생 관리 효율화, 비용 대비 효과 |
| 2 | 학교 교사 (정규/방과후) | 바로 쓸 수 있는 수업 도구, 진도 관리 자동화 |
| 3 | 학부모 | 아이의 학습 현황 투명성, 개인정보 보호 신뢰 |

## 4. 타이포그래피

### 4.1 Font Family 선택

**주 서체**: **Noto Sans KR** (Google Fonts)
- 선택 이유: 한글/영문 혼합 가독성 최상, weight 다양(300~900), 둥근 형태로 눈의 피로 최소화, 인쇄 시 선명도 우수
- 인쇄용 대체: 본고딕 (Noto Sans CJK KR) OTF 사용

**보조 서체 (영문 브랜드명)**: **Nunito** 또는 **Poppins**
- 선택 이유: 둥근 형태의 기하학적 산세리프, 따뜻하고 친근한 인상, "Let's Coding" 로고 텍스트에 사용

**코드 서체 (코드 예시 영역)**: **JetBrains Mono**
- 선택 이유: 프로그래밍 맥락 전달, 기존 플랫폼과 일관성 유지

### 4.2 서체 크기 체계 (인쇄 기준)

| 용도 | 서체 | Weight | 크기 | 행간(leading) | 자간 |
|------|------|--------|------|---------------|------|
| 표지 메인 타이틀 | Noto Sans KR | Bold (700) | 24pt | 32pt | -20 |
| 표지 서브 타이틀 | Noto Sans KR | Medium (500) | 12pt | 18pt | 0 |
| 섹션 타이틀 | Noto Sans KR | Bold (700) | 14pt | 20pt | -10 |
| 섹션 서브 타이틀 | Noto Sans KR | Medium (500) | 10pt | 16pt | 0 |
| 기능 카드 제목 | Noto Sans KR | SemiBold (600) | 10pt | 14pt | -10 |
| 본문 텍스트 | Noto Sans KR | Regular (400) | 8pt | 13pt | 0 |
| 보조 설명 | Noto Sans KR | Regular (400) | 7pt | 11pt | 0 |
| 숫자 강조 (통계) | Nunito/Poppins | ExtraBold (800) | 20pt | - | -30 |
| 브랜드명 "Let's Coding" | Nunito | Bold (700) | 14pt | - | -10 |
| 코드 예시 | JetBrains Mono | Regular (400) | 7pt | 10pt | 0 |
| QR 코드 라벨 | Noto Sans KR | Medium (500) | 7pt | 10pt | 0 |
| 연락처 정보 | Noto Sans KR | Regular (400) | 7.5pt | 12pt | 0 |

### 4.3 읽기 편안함을 위한 타이포 규칙

- 본문 최소 크기: **7pt** (이보다 작으면 가독성 저하)
- 본문 행간: 글자 크기의 **1.5~1.7배** 유지
- 한 줄 최대 글자 수: **25~30자** (패널 폭 99mm 기준)
- 단락 간 여백: 행간의 **1.5배**
- 강조 표현: Bold weight 변환 또는 Warm Teal 색상 적용 (밑줄 미사용)
- 한글-영문 혼합 시: 영문 baseline을 한글에 맞춰 미세 조정

## 10. 디자인 톤 & 무드 요약

| 항목 | 지시 |
|------|------|
| **전체 분위기** | 따뜻하고 밝은 에듀테크. "아이의 성장을 함께 응원하는" 느낌 |
| **색상 키워드** | 아이보리 화이트, 민트, 크림, 코랄 악센트 — 차갑지 않은 Light 테마 |
| **폰트 키워드** | 둥글고 부드러운 산세리프, 가독성 최우선, 눈의 피로 최소화 |
| **사진/이미지** | 실제 플랫폼 화면 위주, 목업 프레임으로 세련되게, 따뜻한 톤 보정 |
| **피해야 할 것** | 차가운 블루 단독 사용, 날카로운 모서리, 과도한 그래디언트, 빽빽한 텍스트 |
| **참고 브랜드 무드** | 클래스101 (따뜻한 에듀 톤), 엘리스 (깔끔한 코딩 교육), 노션 (미니멀 + 따뜻함) |
| **여백(White Space)** | 충분히 확보. 각 패널 상하좌우 최소 8mm 마진. 정보 밀도보다 가독성 우선 |
| **접힘 경험** | 표지(F)로 호기심 유발 → 펼치면 기능+학습흐름+교사기능이 한눈에 → 뒤집으면 학부모 안심+연락처 |

## 11. 읽는 순서 (사용자 경험 흐름)

```
1단계: 표지(Panel F)를 본다
       → "코딩 학습 플랫폼이구나" + 신뢰 수치 확인
       → 호기심 발생

2단계: 리플릿을 펼친다
       → Panel A(핵심 기능 4가지)를 왼쪽부터 읽는다
       → Panel B(학습 흐름)로 "아이가 이렇게 배우는구나" 이해
       → Panel C(교사/기관 기능)에서 "관리도 편하겠다" 확신
       → CTA 버튼 "도입 문의하기" 발견

3단계: 리플릿을 뒤집는다
       → Panel D(학부모 안심)로 신뢰 강화
       → Panel E(연락처/QR)에서 바로 행동 가능
       → QR 스캔 또는 전화/이메일로 문의
```
