## 1. 프로젝트 개요

### 제품명
**Let's Coding & Play** (렛츠코딩앤플레이)

### 서비스 URL
- **랜딩페이지**: https://letscoding.co.kr (이 프로젝트)
- **플랫폼**: https://python.letscoding.co.kr
- **문의 이메일**: contact@letscoding.co.kr

### 한 줄 설명
학원/학교를 위한 AI 기반 파이썬 코딩 교육 플랫폼

### 상세 설명
학생들이 게이미피케이션된 환경에서 파이썬 알고리즘을 학습하고, 선생님은 실시간으로 진행 상황을 모니터링할 수 있는 올인원 교육 솔루션입니다. AI 튜터가 개인 맞춤형 힌트와 피드백을 제공하고, 동영상 문법 지도안과 타자연습 등 다양한 학습 도구를 통해 교육 효율을 극대화합니다.

### 핵심 기능 요약

| 기능 | 설명 |
|------|------|
| 온라인 코드 에디터 | 브라우저에서 바로 Python 코드 작성 및 실행 (CodeMirror) |
| 자동 채점 시스템 | 테스트케이스 기반 즉각적인 정답 확인 |
| AI 튜터 (Claude AI) | 힌트, 코드 리뷰, 에러 분석, 개념 설명, 대화형 튜터링 |
| 별 시스템 & 픽셀아트 | 학습 성취도 시각화 (은색별/금색별), 500문제 픽셀아트 |
| 타자연습 | 100스테이지 Python 코딩 표현식 타자 게임 + 일일 랭킹 |
| 모두의 문제 | 학생이 직접 문제를 출제하고 AI가 난이도를 자동 분류 |
| 동영상 문법 지도안 | Bunny Stream 기반 문법별 동영상 강의 |
| AI 학습 성과 보고서 | AI가 학생 학습 성과를 분석하여 선생님용 보고서 생성 |
| 학부모 피드백 공유 | 보안 링크를 통한 학부모 열람 (로그인 불필요) |
| 관리 대시보드 | 기관/클래스/학생 통합 관리, 실시간 로그인/학습량 모니터링 |
| 멀티테넌시 | 여러 기관/학원 독립적 운영 |

## 2. 타겟 고객

### Primary: 코딩 학원 원장/운영자
- **페인 포인트**: 학생 관리 어려움, 수업 품질 표준화, 강사 의존도
- **니즈**: 효율적인 학생 관리, 자동화된 채점, 학습 현황 파악

### Secondary: 코딩 강사/선생님
- **페인 포인트**: 수업 준비 시간, 개별 피드백 한계, 진도 관리
- **니즈**: 커리큘럼 제공, AI 보조, 실시간 모니터링, 동영상 자료

### Tertiary: 학부모
- **페인 포인트**: 자녀 학습 현황 파악 어려움
- **니즈**: 투명한 진행 상황, AI 학습 성과 보고서

## 3. 핵심 강점

> 이 섹션은 랜딩페이지에서 학생/선생님 관점 탭 또는 2단 레이아웃으로 표현

### 학생 관점

| 강점 | 설명 | 상세 기능 |
|------|------|----------|
| AI 인공지능 튜터 | 24시간 언제든 도움받을 수 있는 개인 튜터 | 단계별 힌트, 코드 리뷰, 에러 분석, 개념 설명, 대화형 채팅 |
| 단계별 문법 학습 | 출력부터 함수까지 체계적인 10단계 커리큘럼 | 10개 카테고리, 카테고리별 문제, 동영상 문법 강의 |

### 선생님 관점

| 강점 | 설명 | 상세 기능 |
|------|------|----------|
| AI 학습 성과 보고서 | AI가 학생별 학습 데이터를 분석하여 보고서 자동 생성 | 학부모 공유 링크(로그인 불필요), 개별 메모, 피드백 이력 |
| 학생 코딩 학습 관리 | 실시간 로그인 상태 및 하루 학습량 체크 | 클래스별 현황, 진행률, 최근 활동, AI 사용 내역 추적 |
| 동영상 문법 지도안 | 문법 주제별 동영상 강의 자료 제공 | Bunny Stream 기반, 문제 풀이 페이지에서 바로 시청 가능 |

## 4. 핵심 가치 제안 (Value Proposition)

### 메인 카피
> **"AI와 함께하는 스마트한 코딩 교육"**
>
> 학생은 재미있게, 선생님은 편리하게, 학원은 효율적으로

### 서브 카피
> 자동 채점 x AI 튜터 x 실시간 관리 = 코딩 교육의 새로운 기준

### 3가지 핵심 USP

1. **AI 튜터**: 24시간 개인 맞춤형 학습 지원 + 선생님용 AI 보고서
2. **게이미피케이션**: 별 시스템, 픽셀아트, 타자연습 랭킹으로 학습 동기 유발
3. **올인원 관리**: 기관-클래스-학생 통합 대시보드 + 학부모 피드백 공유

### 6.6 How It Works (작동 방식)

### 8.4 도입문의 연동 (혼합 방식: Supabase DB + Resend 이메일)

**아키텍처**:
```
[랜딩페이지 폼] → POST /api/contact (Vercel Serverless Function)
                        ↓
                ┌───────┴───────┐
                ↓               ↓
        Supabase DB 저장    Resend 이메일 전송
      (관리자 대시보드 확인)  (즉시 알림)
```

**장점**:
- CORS 문제 없음 (같은 도메인의 서버리스 함수)
- python-algorithm-ver2 코드 수정 불필요
- 같은 Supabase DB 공유 → 관리자 대시보드에서 문의 내역 확인 가능
- Resend로 즉시 이메일 알림 → 빠른 응대

**Supabase 테이블** (`contact_inquiries`):

```sql
CREATE TABLE contact_inquiries (
  id BIGSERIAL PRIMARY KEY,
  phone VARCHAR(20) NOT NULL,
  message TEXT NOT NULL,
  status VARCHAR(20) DEFAULT 'pending',  -- pending / contacted / closed
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

**서버리스 함수** (`api/contact.ts`):

```typescript
// POST /api/contact
// Request body: { phone: string, message: string }
// Response: { success: boolean, error?: string }

// 1. 요청 유효성 검증 (전화번호, 문의내용 필수)
// 2. Supabase에 contact_inquiries 테이블에 저장 (service_role key 사용)
// 3. Resend API로 알림 이메일 전송
//    - 발신: contact@letscoding.co.kr (RESEND_FROM_EMAIL)
//    - 수신: contact@letscoding.co.kr
//    - 제목: "[도입문의] {전화번호}"
//    - 본문: 전화번호 + 문의 내용 (HTML 템플릿)
// 4. DB 저장 성공이 우선, 이메일 실패 시에도 성공 응답 (이메일은 보조 알림)
```

**참고**: 프론트엔드에서 직접 API 키를 사용하지 않고, Vercel Serverless Function을 통해 키를 보호합니다.
